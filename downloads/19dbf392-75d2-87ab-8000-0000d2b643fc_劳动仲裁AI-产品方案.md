# 劳动仲裁AI智能体 - 产品方案文档

> 基于12维度深度研究成果整理 | 版本: MVP v1.0 | 日期: 2026-04-24

---

## 1. 产品定位

### 1.1 一句话定义
**"律师背书的劳动仲裁全流程AI助手"** —— 不只是问答，而是从纠纷评估、证据采集、文书生成到流程指导的完整解决方案。

### 1.2 核心价值主张
- **对劳动者**: 将3000-8000元的律师咨询费降为0，让每个人都能获得专业级法律支持
- **对律师/律所**: 获取精准案源，提升品牌公信力
- **差异化**: 全流程服务（非仅问答）+ 律师实名背书（非匿名AI）

### 1.3 市场机会
| 指标 | 数据 |
|------|------|
| 2024年劳动仲裁案件 | 425.7万件 |
| 涉及劳动者 | 454.9万人 |
| 律师咨询费区间 | 3000-8000元 |
| 劳动者自诉比例 | 约70%（无力请律师） |
| 竞品细分覆盖 | 浅（通义法睿、法行宝等偏向通用法律） |

## 2. 目标用户画像

### 2.1 核心用户（Primary）
**"小张"** —— 25-35岁普通劳动者，月薪5K-15K
- 遭遇违法解除、欠薪、未缴社保等常见问题
- 完全不懂法律流程，不会写申请书
- 想维权但请不起律师（律师费可能超过争议金额）
- 对AI有基本信任但担心"胡说八道"
- 核心诉求：告诉我该做什么、帮我写好材料、告诉我能拿多少钱

### 2.2 次要用户（Secondary）
**"李律师"** —— 执业3-10年的劳动法律师
- 需要精准案源（已梳理清楚案情、证据充分的当事人）
- 愿意为平台背书换取品牌曝光和案源
- 不担心AI抢饭碗（仲裁后的一审二审仍需律师）

## 3. MVP核心功能范围（Phase 1）

### 3.1 功能取舍原则
- **必须做**: 用户最痛、技术可行、律师背书价值高
- **不做**: 需要外部语料库API、需要复杂数据源、非核心流程

### 3.2 MVP功能清单（6大模块）

#### 模块A: 智能案情评估（入口）
**功能**: 多轮对话收集纠纷信息，输出评估报告
- 采集: 入职时间、岗位、薪资结构、纠纷类型、证据情况
- 输出: 胜率预估（高/中/低）、可主张权益清单、金额估算（含计算逻辑）
- 设计要点: 用通俗语言解释法律概念，避免法条堆砌

#### 模块B: 证据采集指导（差异化核心）
**功能**: 根据案情生成个性化证据清单，指导用户收集
- 输出: "您需要准备以下证据：1. 劳动合同（证明劳动关系）2. 工资流水..."
- 进阶: 证据有效性判断（"这张截图不够，需要银行流水"）
- 设计要点: 分"必备/加分/可有可无"三级，降低用户焦虑

#### 模块C: 仲裁申请书生成（高频刚需）
**功能**: 基于采集信息自动生成仲裁申请书
- 输出: 标准格式Word/PDF文档，含所有必填字段
- 包含: 当事人信息、仲裁请求、事实与理由、证据清单
- 设计要点: 允许用户逐项确认/修改，不是一次性输出

#### 模块D: 流程导航（降低门槛）
**功能**: 仲裁全流程可视化指引
- 输出: 步骤图（申请→受理→开庭→裁决→执行）
- 提醒: 每个节点的时限（"收到受理通知后5日内提交证据"）
- 地域适配: 各地仲裁委地址、联系方式、特殊要求（待Phase 2）

#### 模块E: 金额计算器（实用工具）
**功能**: 赔偿金/补偿金/加班费自动计算
- 输入: 工作年限、月平均工资、纠纷类型
- 输出: 明细计算表（N、N+1、2N等）
- 约束处理: 社平工资3倍封顶提醒（需用户手动输入当地数据或全国估算）

#### 模块F: 律师背书展示（信任机制）
**功能**: 律师实名背书+联系方式展示
- 展示: 合作律师头像、执业证号、律所、专长领域
- 入口: "如需代理，可联系XX律师"（案源转化）
- 设计要点: 不强制推销，在用户明确表达需求后推荐

### 3.3 MVP不做（Phase 2+）
| 功能 | 原因 |
|------|------|
| 在线提交仲裁申请 | 各地系统不统一，需RPA或人工 |
| 实时判例检索 | 无外部语料库API |
| 自动证据OCR识别 | 技术复杂，Phase 2实现 |
| 各地社平工资自动获取 | 无统一数据源，需设计更新策略 |
| 多语言支持 | 聚焦中文用户 |
| 视频/语音咨询 | 成本过高 |

## 4. 用户旅程设计

### 4.1 主流程（Happy Path）
```
用户进入 → 选择纠纷类型（欠薪/违法解除/未缴社保等）
    ↓
多轮对话采集案情（5-10个问题）
    ↓
生成【案情评估报告】（胜率+权益清单+金额估算）
    ↓
用户确认"我要申请仲裁"
    ↓
生成【证据清单】（个性化）→ 用户上传/确认证据
    ↓
生成【仲裁申请书】（Word/PDF下载）
    ↓
展示【流程导航】（下一步做什么）
    ↓
可选：联系合作律师（案源转化）
```

### 4.2 关键交互设计
- **进度条**: 每个环节显示"已完成X%，还剩Y步"
- **随时保存**: 用户可中断，下次继续
- **人工兜底**: 每个环节提供"这步我不太明白，帮我看看"入口

## 5. 数据策略（应对约束）

### 5.1 社平工资数据（已知约束：无统一数据源）
**方案**: 三层策略
1. **基础层**: 内置全国主要城市2024年社平工资表（手动整理Top 50城市）
2. **用户层**: 允许用户手动输入（"如果您知道当地数据，请输入"）
3. **估算层**: 基于用户所在城市等级，给出估算值并明确标注"估算值，仅供参考"

### 5.2 法律知识来源（已知约束：完全依赖大模型）
**方案**: 
- 模型自带知识（劳动法、劳动合同法、仲裁法）
- Prompt工程: 要求模型明确标注法律依据和不确定性
- 免责声明: 每个输出页脚标注"AI生成，仅供参考，不构成法律意见"

### 5.3 数据更新策略
- 法律条文变更: 依赖模型训练数据时效性，Prompt要求标注"截至XX年"
- 社平工资更新: 每季度手动更新Top 50城市数据（低成本）

## 6. 律师背书机制

### 6.1 合作模式
| 模式 | 说明 |
|------|------|
| 品牌背书 | 律师/律所logo展示，平台权威性提升 |
| 内容审核 | 律师定期审核AI生成内容的准确性（可选） |
| 案源转化 | 用户选择"联系律师"，信息推送给合作律师 |

### 6.2 律师展示设计
- 首页: "本服务由XX律师事务所提供专业支持"
- 结果页: "以上内容基于公开法律知识，如需代理请联系XX律师"
- 律师详情: 头像、姓名、执业证号、专长、成功案例数、用户评分

## 7. 信任机制设计

### 7.1 三层信任建设
1. **透明层**: 每个AI回答标注"依据《劳动合同法》第X条"，不确定时标注"此问题建议咨询律师"
2. **可控层**: 用户可逐项修改AI生成的任何内容（申请书、证据清单）
3. **人工层**: 明确提供"转人工"入口，不隐藏、不阻碍

### 7.2 风险防控
- **内容标识**: AI生成内容明确标注（监管合规）
- **免责声明**: 每页底部固定区域展示
- **敏感词过滤**: 禁止提供虚假证据指导等违规建议
- **算法备案**: 产品上线前完成网信办算法备案（监管要求）

## 8. 技术实现建议（给小号🦐的输入）

### 8.1 技术选型建议
| 层级 | 建议方案 | 理由 |
|------|----------|------|
| 前端 | Next.js + Tailwind | 快速开发、SEO友好、部署便捷 |
| 后端 | Next.js API Routes / 轻量Node | MVP最小成本 |
| AI能力 | 大模型API（Kimi/Claude/DeepSeek） | 完全依赖大模型，无需自建 |
| 数据库 | SQLite / PostgreSQL | 用户数据、进度保存 |
| 文件存储 | 本地/云存储 | 生成Word/PDF |
| 部署 | Vercel / 自有服务器 | 低成本 |

### 8.2 核心Prompt设计思路
- **案情采集Prompt**: 结构化多轮对话，逐步收集信息
- **评估报告Prompt**: 输出JSON结构化数据（胜率、权益清单、金额）
- **申请书Prompt**: 输出标准法律文书格式，含所有必填字段
- **证据清单Prompt**: 基于案情类型匹配证据模板

### 8.3 关键实现难点
1. **长文本生成稳定性**: 仲裁申请书需要长文本输出，需设计分段生成+组装策略
2. **多轮对话状态管理**: 案情采集可能中断，需保存对话状态
3. **Word/PDF生成**: 可用docx.js或Pandoc实现模板填充
4. **金额计算准确性**: 需Prompt明确计算规则，并添加人工复核机制

## 9. 商业模式初探（MVP后可验证）

### 9.1 收入路径
| 阶段 | 模式 | 说明 |
|------|------|------|
| MVP期 | 免费 | 积累用户和案例 |
| 成长期 | 增值功能付费 | 高级文书模板、优先律师对接 |
| 成熟期 | SaaS订阅 / 案源分成 | 面向律所的案源平台 |

### 9.2 成本结构（MVP期）
- 大模型API调用: 主要成本（按量付费）
- 服务器: Vercel免费档或低价VPS
- 律师合作: 初期可0成本（以品牌曝光换背书）

## 10. MVP验收标准

### 10.1 功能验收
- [ ] 用户可通过多轮对话完成案情采集
- [ ] 生成完整的案情评估报告（胜率+权益+金额）
- [ ] 生成个性化的证据采集清单
- [ ] 生成标准格式的仲裁申请书（可下载Word/PDF）
- [ ] 展示完整的仲裁流程导航
- [ ] 律师背书信息正确展示

### 10.2 体验验收
- [ ] 完整流程耗时 < 15分钟
- [ ] 任意步骤可保存/中断/继续
- [ ] 每个AI输出有法律依据标注或不确定性提示
- [ ] 移动端可用

### 10.3 合规验收
- [ ] 所有AI生成内容有明确标识
- [ ] 免责声明完整展示
- [ ] 敏感内容有拦截机制
- [ ] 完成算法备案（上线前）

## 11. 迭代路线图

### Phase 1 (MVP, 4-6周)
核心6模块上线，基础律师背书，最小成本验证

### Phase 2 (2-3月后)
- 各地仲裁委信息库（地址、联系方式、特殊要求）
- 在线证据上传与OCR识别
- 律师在线接单系统
- 用户案例积累与展示

### Phase 3 (6月后)
- 判例检索（接入外部API）
- 多场景扩展（工伤、社保、公积金）
- 企业端（HR合规自查）

## 12. 风险评估

| 风险 | 等级 | 应对策略 |
|------|------|----------|
| AI生成内容不准确 | 高 | 律师审核机制 + 不确定性标注 + 免责声明 |
| 用户误用导致败诉 | 高 | 全流程风险提示 + 明确标注"仅供参考" |
| 律师合作不稳定 | 中 | 签约多家律所，不绑定单一律师 |
| 监管合规问题 | 中 | 算法备案 + 内容标识 + 数据安全 |
| 竞品快速跟进 | 低 | 律师背书 + 全流程服务是差异化壁垒 |
