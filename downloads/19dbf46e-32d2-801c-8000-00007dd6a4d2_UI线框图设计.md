# 劳动仲裁AI智能体 - UI线框图设计

> 文本版线框图，供前端开发参考。所有尺寸为近似值，以实际开发为准。

---

## 设计原则

1. **移动端优先**：60%用户用手机，所有页面先适配移动端
2. **信任感**：蓝色系主色调（法律、专业、可靠），白色背景，大量留白
3. **渐进式引导**：一步一步来，不让用户一次性面对复杂表单
4. **即时反馈**：AI思考时显示动画，操作后立即响应

## 色彩方案

```
主色：#2563EB（蓝色 - 专业、信任）
辅助色：#10B981（绿色 - 成功、完成）
警告色：#F59E0B（橙色 - 提示、注意）
危险色：#EF4444（红色 - 错误、删除）
文字主色：#1F2937（深灰）
文字次色：#6B7280（中灰）
背景色：#F9FAFB（浅灰）
卡片背景：#FFFFFF（白色）
分割线：#E5E7EB
```

### 关键交互

1. **消息气泡**
   - 用户消息：白色背景，右侧对齐
   - AI消息：浅蓝色背景（#EFF6FF），左侧对齐
   - 系统消息（选项按钮）：居中，浅灰色背景

2. **选项按钮**
   - 纠纷类型选择：圆角矩形按钮，2列网格
   - 选中状态：蓝色背景 + 白色文字
   - 未选中：白色背景 + 蓝色边框

3. **推理过程**
   - 默认收起，点击"查看推理过程"展开
   - 展开后显示步骤编号（①②③④）
   - 计算部分用等宽字体展示公式

4. **行动卡片**
   - 分析结果下方显示两个大按钮
   - "生成证据清单"和"生成仲裁申请书"
   - 点击后跳转对应页面

5. **输入框**
   - 固定在底部
   - 左侧输入框，右侧发送按钮
   - 支持文字输入（语音输入V2）

## 动画与过渡

1. **消息出现**：从底部滑入，淡入，200ms
2. **选项按钮出现**：依次淡入，间隔50ms
3. **推理过程展开**：高度从0展开，300ms，ease-out
4. **页面切换**：滑动过渡，左进右出
5. **加载状态**：圆点跳动动画，1s循环
6. **进度条更新**：宽度变化，300ms，ease-in-out
